# SpecClarify Result

## Confirmed

- User wants: a small app where users can sign up, invite friends, and maybe get points later.

## Missing

- target users
- core features
- scope / MVP boundary
- auth and roles
- data persistence
- admin / operations
- non-functional constraints
- timeline

## Must Ask

- Who are the primary users or target audience?
- What are the core features for the first version?
- What is the minimum viable scope for the first release?

## Assumptions

- Assume a web-based MVP unless otherwise specified.
- Assume a single end-user role unless multiple roles are explicitly mentioned.
- Assume deferred features are out of MVP scope unless explicitly prioritized.
- Assume minimal persistence (e.g. user profiles) until data model is specified.

## Risks

- Scope creep if MVP boundary is not defined.
- Integration complexity if auth or external services are assumed but unspecified.
- Time pressure may conflict with completeness of first release.
- Deferred features ('maybe later') can create future rework.
