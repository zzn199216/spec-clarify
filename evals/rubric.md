# SpecClarify Evaluation Rubric

Manual evaluation criteria for clarified outputs. Score each 1–5.

## 1. Captures main goal

Does the output correctly identify and state the primary goal from the raw requirement?

- **1**: Misses or distorts the main goal
- **3**: Partially captures the goal
- **5**: Clearly and accurately states the main goal

## 2. Avoids missing critical requirements

Does the output surface important requirement areas that are absent or vague?

- **1**: Ignores major gaps (users, scope, auth, etc.)
- **3**: Identifies some gaps
- **5**: Proactively identifies critical missing areas

## 3. Avoids over-assumption

Does the output avoid assuming details that were not stated?

- **1**: Heavily assumes unstated details
- **3**: Some over-assumption
- **5**: States assumptions explicitly; does not invent requirements

## 4. Produces execution-friendly output

Is the output structured and concrete enough for implementation?

- **1**: Unusable for implementation
- **3**: Partially actionable
- **5**: Clear, structured, ready for implementation planning

## 5. Separates MVP vs later scope appropriately

Does the output distinguish what belongs in the first release vs later?

- **1**: No separation; scope is unclear
- **3**: Partial separation
- **5**: Clear MVP vs out-of-scope / later
